Мы используем стену для тренировки стрелков на стрельбище, в которую надо попасть с разных позиций. У стрелков нет проблем с вертикальным отклонением, поэтому мы можем упростить модель применяемую в данной задаче. Мы разместили камеру сверху и будем использовать прямоугольную систему координат, для описания позиций. Нам известны координаты краев стены и точки выстрела. Также известна точка, где находится пуля спустя какое-то время (точка траектории). Зная эти данные нам нужно определить результат выстрела.

Результат выстрела определяется в зависимости от точки попадания в стене, где самый центр - это 100 очков. Результат уменьшается, чем дальше попадание от центра. Попадание в край стены - это 0 очков. Промах - это -1 очко. Результат должен быть округлен к ближайшему целому числу.

Входные данные: Четыре аргумента. Два края стены, точка выстрела и вторая точка траектории пули, как кортежи (tuple) с двумя целыми числами в каждом.

Выходные данные: Результат, как целое число от -1 до 100.

Примеры: 

shot((2, 2), (5, 7), (11, 2), (8, 3)) == 100
shot((2, 2), (5, 7), (11, 2), (7, 2)) == 0
shot((2, 2), (5, 7), (11, 2), (8, 4)) == 29
shot((2, 2), (5, 7), (11, 2), (9, 5)) == -1
shot((2, 2), (5, 7), (11, 2), (10.5, 3)) == -1

Как это используется: Методы из данной задачи могут быть использованы для моделирования траекторий или определения видимости объектов с различных точек.

Предусловия:
all(all(0 < i < 100 for i in coor) for coor in args)
wall1, wall2 and shot_point are not collinear.
